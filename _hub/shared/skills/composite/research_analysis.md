# Research & Analysis 複合技能

## 用途

Ori 在進行競品研究、市場趨勢分析、選品候選調查時調用此技能。
目標：產出**有依據、有結構、可直接被其他 agent 使用**的研究報告。

---

## 一、研究品質標準

### 好的研究報告有什麼

| 標準 | 說明 |
|------|------|
| **來源可信** | 有明確出處，不是臆測 |
| **數據具體** | 有數字，不是「大約」、「可能」 |
| **結論直接** | 說清楚「所以呢」，不只是列資料 |
| **可操作** | 後面的 agent 拿到可以直接用 |
| **時間標記** | 資料是什麼時候的，是否還有效 |

### 常見研究錯誤（避免）
- ❌ 只收集資料，不給結論
- ❌ 資料來源不明
- ❌ 沒有說明與現有系統的關係
- ❌ 用「可能」、「也許」代替有根據的判斷

## 二、競品分析框架

### 分析維度
```
1. 基本資料
   - 競品名稱、帳號、定位
   - 受眾規模（追蹤數、互動率）

2. 內容策略
   - 發文頻率
   - 主要內容類型（教育/娛樂/銷售）
   - 最高互動的貼文類型

3. 差異化機會
   - 他們做了什麼我們沒做？
   - 他們的弱點是什麼？
   - 有什麼他們忽略的受眾需求？

4. 結論建議
   - 我們應該跟進什麼？
   - 我們應該差異化什麼？
```

### 輸出格式
```markdown
## 競品分析：[競品名稱]
分析日期：YYYY-MM-DD

### 基本資料
- 平台帳號：
- 追蹤數：
- 平均互動率：

### 內容特徵
- 發文頻率：
- 主要類型：
- 爆文特徵：

### 機會點
1. [我們可以做但他們沒做的]
2. [他們的明顯弱點]
3. [受眾需求缺口]

### 建議行動
→ 給 Lala：[選題建議]
→ 給 Craft：[文案風格參考/對比]
```

## 三、市場趨勢分析 SOP

### 資料來源優先級
1. **Perplexity 即時搜尋**（已整合）— 最新趨勢
2. **Threads/IG 爆文分析**— 社群實際反應
3. **電商平台數據**（蝦皮、Shopee 熱銷）— 商業驗證
4. **Google Trends**— 搜尋量趨勢

### 分析流程
```
輸入：主題或商品類別
  ↓
1. Perplexity 搜尋最新資訊
2. 識別關鍵趨勢信號（上升/下降/平穩）
3. 交叉驗證（至少 2 個來源）
4. 評估商業潛力（量 × 利潤）
5. 輸出結構化報告
```

## 四、選品研究 SOP

### 候選商品評估維度
| 維度 | 說明 | 權重 |
|------|------|------|
| 市場需求 | 有沒有人在找這個？ | 30% |
| 毛利空間 | 賣了能賺多少？ | 25% |
| 競爭程度 | 市場有多擠？ | 20% |
| 內容潛力 | 這個商品能說什麼故事？ | 15% |
| 供應穩定性 | 能持續供貨嗎？ | 10% |

### 輸出給選品系統
研究結果輸出後通知 Sage 進行評分，再轉入 `fl_products`。

---

## 五、研究深度判斷

| 情境 | 深度 | 預計時間 |
|------|------|---------|
| 日常選題素材蒐集 | 淺（掃描 + 摘要） | 5-10 分鐘 |
| 新品類評估 | 中（多來源交叉） | 20-30 分鐘 |
| 完整競品報告 | 深（全面分析） | 1 小時+ |

**原則：匹配任務規模，不要每次都做最深的研究。**
