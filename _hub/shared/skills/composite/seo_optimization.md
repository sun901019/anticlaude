# SEO / GEO Optimization 複合技能

## 用途

Sage 在進行品牌搜尋分析、內容可見度評估、SEO 建議時調用此技能。
重點：同時考慮**傳統搜尋引擎（Google）**和**AI 搜尋平台（Perplexity、ChatGPT Search）**的可見度。

---

## 一、GEO vs SEO 的差異

| 維度 | 傳統 SEO | GEO（AI 搜尋優化）|
|------|---------|-----------------|
| 目標 | Google 排名 | 被 AI 引用/摘要 |
| 評分標準 | 關鍵字密度、反向連結 | E-E-A-T、可引用性 |
| 內容長度 | 越長越好 | 夠用就好，段落要獨立 |
| 結果形式 | 搜尋結果頁排名 | AI 回答中被直接引用 |

**2026 年的重點：GEO 比 SEO 更重要。**
越來越多人直接問 AI，不點進搜尋結果。

## 二、E-E-A-T 評估框架（滿分 100）

### Experience（體驗）— 25 分
內容是否來自**親身經歷**？

強信號：
- 第一人稱描述：「我測試過...」、「實際用了三個月...」
- 原創數據或調查結果
- 具體案例 + 真實數字

弱信號（扣分）：
- 只轉述別人說的
- 模糊語言：「據說」、「可能」、「有研究指出」
- 沒有具體使用過程

### Expertise（專業）— 25 分
內容是否展現**深度知識**？

強信號：
- 技術深度適當（不是表面介紹）
- 有方法論說明（如何得出結論）
- 數據支撐論點

弱信號：
- 只有表面層次
- 沒有說明推論過程
- 廣但淺

### Authoritativeness（權威）— 25 分
是否被**他人認可**？

強信號：
- 被其他網站引用
- 在媒體上被提到
- 對特定主題有完整的內容群

弱信號：
- 孤立的單篇文章
- 沒有外部驗證

### Trustworthiness（可信）— 25 分
是否**透明、準確**？

強信號：
- 有聯絡資訊
- 有清楚的發佈時間
- 廣告/業配有明確標示
- 資訊準確（可驗證）

## 三、AI 引用優化（Citability）

### 讓 AI 更容易引用你的內容

**段落結構原則：**
- 每段 2-4 句
- 每段只有一個概念
- 第一句就說重點
- 段落要能獨立存在（抽出來還是有意義）

**標題原則：**
- 用問句形式的標題（AI 偏好直接回答問題）
- 「如何...」、「什麼是...」、「為什麼...」
- 描述性，不要「第三章」這種結構

**內容新鮮度：**
- 有明確的發佈/更新日期
- 定期更新舊內容
- 避免使用「最新」、「近期」等相對時間

## 四、Sage 的 SEO 分析 SOP

### 品牌可見度分析
```
1. 搜尋品牌名稱（@sunlee._.yabg、Sun Lee）
2. 搜尋核心主題關鍵字（台灣選品、Threads 電商等）
3. 查看 AI 搜尋（Perplexity）是否有引用
4. 評估當前 E-E-A-T 得分
5. 找出 3 個最容易改善的點
```

### 內容機會分析
```
1. 找出目前沒有內容覆蓋的關鍵字
2. 找出競品有排名但我方沒有的主題
3. 找出可以加強 E-E-A-T 的現有內容
4. 輸出優先級建議給 Lala 納入選題
```

### 輸出格式
```markdown
## SEO 快速評估報告
日期：YYYY-MM-DD

### 品牌能見度
- Google 搜尋：[結果數 / 排名位置]
- AI 搜尋引用：[有/無，具體例子]

### E-E-A-T 評分
| 維度 | 得分 | 主要問題 |
|------|------|---------|
| Experience | /25 | ... |
| Expertise | /25 | ... |
| Authoritativeness | /25 | ... |
| Trustworthiness | /25 | ... |

### 前三名改善建議
1. [最高優先級]
2. [次優先級]
3. [長期建議]

### 給 Lala 的選題建議
- [關鍵字機會 1]
- [關鍵字機會 2]
```

## 五、低品質 AI 內容的警告信號

發現以下模式時，Sage 應通知 Craft 修正：

| 信號 | 描述 |
|------|------|
| 通用開場 | 「在當今快節奏的世界...」 |
| 無原創洞察 | 只重組常見資訊 |
| 沒有第一人稱 | 沒有個人觀點或經歷 |
| 結構完美但空洞 | 標題工整但內容淺 |
| 模糊結論 | 「效果因人而異」沒有說清楚哪些因素 |
