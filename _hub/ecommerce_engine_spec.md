# 🛒 AntiClaude 擴充模組：電商選品決策引擎 (E-commerce Product Selection Engine)

> **定位**：這不是單純的商品分析工具，而是將「資深電商營運的決策邏輯」轉化為自動化的決策引擎。
> **目標**：降低思考成本、減少錯誤決策、建立可複製的選品流程。

---

## 一、系統雙階段流程架構

系統分為「自動探索」與「手動深度分析」兩個主要階段，並將所有資料沉澱為持久化的選品資料庫。

### 階段一：選品機會探索（每週自動化）
**目標**：在雜訊中自動篩選出符合品牌定位且具備潛力的商品靈感。

- **觸發機制**：每週 Cron Job 定時執行。
- **資料來源**：Serper API (Google Search), Google Trends, 或是指定電商平台爬蟲。
- **處理流程**：抓取關鍵字與市場趨勢 → AI (Lala/Sage) 根據品牌調性進行初步過濾。
- **輸出產物**：**《本週選品靈感報告》**
  - 商品名稱與核心關鍵字
  - 為什麼可能有機會（市場缺口/爆點）
  - 市場熱度評估（高/中/低）
  - 品牌定位契合度（1-5 分）
  - 建議行動（強烈建議深度分析 / 建議觀察 / 忽略）

### 階段二：深度選品決策（手動觸發）
**目標**：透過量化數據與質化評價的雙重計算，給出最終的財務與策略建議。

- **觸發機制**：在《選品靈感報告》上點擊「一鍵深度分析」，並輸入 **進貨/運費/包材成本**。
- **處理流程**：AI 進入深度爬蟲與試算模式（拆分為三大核心引擎：市場、評價、財務）。
- **輸出產物**：**《商品深度決策報告》**

---

## 二、深度決策核心引擎設計

### 1. 市場分析引擎 (Market Intelligence)
- 自動抓取目標市場的宏觀數據：
  - **價格區間**：市場最低價 ~ 最高價分佈
  - **銷量中位數**：評估市場天花板
  - **搜尋量**：評估主動需求強度
  - **競品數量**：評估紅海/藍海程度

### 2. 負評痛點引擎 (Negative Review Analyzer) ⭐核心差異化⭐
從 Amazon/Shopee 等平台自動抓取並提煉 1 星與 2 星評論。
- **資料提煉**：不只給原句，而是歸納出「常見負評關鍵字」（例如：品質問題、噪音、不耐用、功能不符、包裝差）。
- **商業價值**：將痛點轉化為行銷賣點（Marketing Hook）或下一代產品的規格改良指南。

### 3. 財務試算與角色鑑定引擎 (Financial & Persona Engine)
根據使用者輸入的成本（進貨 + 運費 + 包材）自動計算財務基線。

#### A. 財務指標計算：
- **總成本** = 進貨 + 運費 + 包材
- **建議售價**（根據市場區間與品牌定位推算）
- **毛利** = 建議售價 - 總成本
- **毛利率** = (毛利 / 建議售價) * 100%
- **CPA 上限** (Cost Per Action) = 可承受的最大廣告成本
- **ROAS 門檻** = 損益兩平所需的廣告投資報酬率

#### B. 商品角色自動判斷邏輯：
系統強制根據「毛利率健康標準」對商品進行策略分類，並檢核是否符合全店 40/40/20 的黃金比例。

| 角色 | 全店建議比例 | 健康毛利率標準 | 判斷邏輯 / 警示機制 |
|------|:---:|:---:|------|
| **引流款** | 40% | ≥ 15% | 靠低價或爆款特質吸客，若毛利率 <15% 觸發「賠本賣流量」警報 |
| **毛利款** | 40% | ≥ 35% | 支撐公司營運利潤的主力，若毛利率 <35% 觸發「利潤過薄」警報 |
| **主力款** | 20% | ≥ 45% | 高單價/高附加價值/獨家規格，若毛利率 <45% 觸發「定位失效」警報 |

## 三、最終輸出與資料庫保存 (The Memory)

### 最終決策報告 (Final Output)
每次深度分析完成後生成的卡片化報告，包含區塊：
1. **市場面** (搜尋量 / 銷量中位數 / 競品數量 / 價格區間)
2. **痛點面** (1-2星評論摘要 / 常見負評關鍵字 / 可優化方向)
3. **財務面** (總成本 / 建議售價 / 毛利與毛利率 / CPA 上限 / ROAS 門檻)
4. **策略面** (商品建議角色 / 判斷原因 / 最終決策建議：強烈建議進貨 / 建議觀察 / 放棄)

### 選品資料庫 (Database Schema)
所有的分析都不是一次性的，必須寫入 SQLite `ecommerce_selections` 表，欄位包含：
- `id`, `product_name`, `keywords`, `analysis_date`
- `market_metrics` (JSON)
- `negative_reviews` (JSON)
- `financials` (JSON)
- `recommended_role` (引流/毛利/主力)
- `decision_status` (待決定/已進貨/已放棄)

**未來價值**：可以比較同檔期不同商品的決策品質，追蹤「當初系統建議進貨的商品，後來真的賣得好嗎？」，形成回溯優化的閉環。

## 四、技術與介面實作層建議

### 技術棧 (Tech Stack)
- **前端介面**：現有的 Next.js 14 儀表板 (`dashboard/`)，新增一區塊 `/ecommerce`。
- **後端 API**：現有的 Python FastAPI (`src/api/`)，新增 `/selection` 路由。
- **資料庫**：現有的 SQLite (`data/anticlaude.db`)，新增選品專屬表。
- **AI 代理**：交由現有 AI-OS 的 `Sage (Analyst)` 與 `Orio (Researcher)` 共同執行。

### 介面設計需求 (UI Requirements)
維持「緊湊、不需華麗」的營運視覺風格：
1. **靈感報告列表 (Weekly Feed)**：類似目前的反直覺快報，但多了「一鍵深度分析」按鈕。
2. **深度分析表單 (Input Form)**：僅需輸入 `[預估進貨成本] [運費] [包材]` 三個欄位即啟動。
3. **決策報告結果頁 (Dashboard)**：呈現上述提到的四個核心區塊報表，毛利率等關鍵數字以紅/綠燈號顯示健康度。
4. **歷史分析庫 (Archive)**：可以過濾查看「已進貨」、「已放棄」、「毛利款」等分類的歷史選品。
