# Skill SOP：選題策略
> 執行者：Lala
> 版本：v2.0（Topic Fit Gate + Similarity Guard + Breakout Cooldown）
> 輸入：ori-to-lala.md（素材清單）+ Threads 最近 7 天數據
> 輸出：lala-to-craft.md（Top 3 主題 + 角度 + 格式建議 + 相似度警告）

---

## 執行步驟

1. 讀取 `ai/handoff/ori-to-lala.md`
2. 讀取 `outputs/threads_metrics/` 最近 7 天，找出高互動主題模式
3. 讀取 `_context/about_me.md` 確認受眾定位

### Step A：Topic Fit Gate（新）
4. 對每個素材執行品牌適配過濾：
   - 品牌核心領域：AI工具 / 職涯 / 個人成長 / 科技趨勢 / 生產力 / 創業
   - **直接剔除**：政治、業配感重、純工程師技術（受眾不是開發者）
   - 適配分數 < 0.5 → 移出候選池，不進入後續評分

### Step B：評分（品牌適配通過後才評分）
5. 依以下維度評分每個素材（1-10）：
   - 受眾匹配度：台灣科技工作者會在意嗎？
   - 時效性：這個話題現在討論熱嗎？
   - 差異性：這個角度別人說過嗎？
   - 品牌適配：符合 AI工具 / 職涯 / 成長方向嗎？
   - **GEO 潛力（新）**：能嵌入可引用實體嗎？有數據或案例支撐嗎？

### Step C：Similarity Guard（新）
6. 對 Top 5 候選主題，對照近 7 天已發貼文：
   - 詞彙重疊率 > 35% → 在 lala-to-craft.md 標注「相似度警告：X%」
   - 詞彙重疊率 > 60% → 換角度或直接換主題
   - 連續 3 天同類型（趨勢 / 工具 / 職涯）→ 強制切換類型

### Step D：Breakout Cooldown（新）
7. 若上一篇貼文互動率 > 5%（高表現貼文）：
   - 冷卻期 48 小時：不發相同類型主題
   - 原因：避免分散高表現貼文的長尾流量

### Step E：選定 Top 3
8. 選出 Top 3，每個附上：
   - 選定角度 + 選擇理由
   - **建議格式**（新）：short / long / thread（基於主題複雜度）
   - **GEO 實體建議**（新）：建議嵌入的 1-2 個術語或數據點
   - 相似度警告（若有）
9. 確認主題比例不失衡（連續 3 天不能都是同類型）
10. 輸出到 `ai/handoff/lala-to-craft.md`

---

## 選題禁區
- 不選純技術教學（受眾不是工程師為主）
- 不選政治話題
- 不選業配感重的工具推薦
- 不選與近 7 天貼文重疊率 > 60% 的主題

```
- 相似度：[X%，若 > 35% 標注警告]

### 主題 2：...
### 主題 3：...
```
