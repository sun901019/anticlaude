## 2. 需要修正的問題

### 2.1 角色門檻不一致

spec 定義：

- < 25%：not_suitable
- 25–40%：traffic（引流款）
- 40–50%：core（毛利款）
- ≥ 50%：profit（主力款）

但現有程式碼（calcPreview + calc_full_cost）用的是：

- < 30%：引流款
- 30–40%：毛利款
- ≥ 40%：主力款

**需統一**。建議採用 spec 的 4 段門檻，並加入 `not_suitable` 警示，因為 <25% 在正常操作環境下根本不該上架。

修正位置：
- `src/ecommerce/router.py` → `calc_full_cost()`
- `dashboard/src/app/ecommerce/page.tsx` → `calcPreview()`

### 2.2 FSS 切換邏輯說明不夠精確

spec 說「selling price > 1000 時比較兩種模式選較便宜」，但其實：

- 在 selling_price = 1000：pct = 1000 × 6% = 60 = fixed（相等）
- 在 selling_price > 1000：fixed NT$60 < pct（百分比制更貴）

所以現有程式碼「> 1000 用 fixed」的邏輯是正確的，不需要真的比較，直接切換即可。
→ spec 文字需修正，程式碼不用改。

### 2.3 逆推定價未完整實作

目前 `calcPreview()` 只給一個 `recommended` 價，但 spec 要求三種定價：

- 引流價（目標 margin 25%，廣告預算 10%）
- 毛利價（目標 margin 40%，廣告預算 15%）
- 主力價（目標 margin 55%，廣告預算 5%）

正確公式：

```
Price = (固定總成本 + 固定費用) / (1 - 蝦皮變動費率 - 目標毛利率 - 廣告預算率)
```

這三個價格都應該在成本預覽中即時顯示，讓操作者直接看到「如果要做引流款，要賣 NT$XXX」。

## 3. 缺少但值得加的功能

### 3.1 市場天花板警示

如果操作者輸入了市場價（低/高），系統應該自動判斷：

- 「引流價 NT$215 < 市場低價 NT$199 → 不建議（進不去市場）」
- 「毛利價 NT$350 > 市場高價 NT$299 → 在市場天花板以上，無法達成」

這讓定價策略即時可見，不用人工比較。

### 3.2 促銷日成本差異提示

成本預覽應該顯示：

「促銷日會多花 NT$X（+2%），本商品在促銷日每件多花 NT$18」

這讓操作者在決定是否參加促銷時有數字依據。

### 3.3 CCB ROI 提示

三種 CCB 方案的固定成本差額應該同時顯示：

```
無 CCB：NT$ 312 / 件
CCB5：NT$ 325 / 件（多 NT$13，但可免部分活動費）
CCB10：NT$ 338 / 件（多 NT$26，免促銷加成）
```

讓操作者在快速進貨決策時能看到數字，不是只能靠感覺。

### 3.4 not_suitable 硬性閘門

當毛利率 < 25% 時，新增商品 modal 應該顯示紅色警示：

「此商品在當前費率下無法達到最低毛利門檻（25%），建議調整售價或成本後再建檔。」

並把「新增商品」按鈕變成 disabled 或 warning 狀態，強制操作者確認。

### 3.5 商品明細 Drawer（重要）

目前編輯是 modal，但應該改成 slide-in drawer，包含以下可收合的區塊：

- A. 基本資料（SKU、名稱、狀態、角色）
- B. 成本與物流（成本、境內運費、集運方式、重量、包材費）
- C. 定價策略（三種定價建議、目標售價、市場價、自訂目標毛利）
- D. 蝦皮環境（CCB、促銷日、備貨天數）
- E. 即時績效（售價、7日銷量、廣告花費、庫存）
- F. AI 建議（補貨、調價、角色確認）

## 6. 定價公式統一規範

所有地方應使用相同公式：

```
建議售價 = (固定總成本 + 固定費用) / (1 - 蝦皮變動費率 - 目標毛利率 - 廣告預算率)

其中：
固定總成本 = 商品成本(TWD) + 集運費(TWD) + 末端配送 + 包材費
固定費用 = FSS_fixed（售價 > 1000 時才加入分子）
蝦皮變動費率 = commission + transaction + ccb + promo + penalty + FSS_pct（售價 ≤ 1000 時）

三種預設目標：
引流款：margin = 25%, ad_rate = 10%
毛利款：margin = 40%, ad_rate = 15%
主力款：margin = 55%, ad_rate = 5%
```

注意：FSS 的處理是難點。
- 售價 ≤ 1000：FSS = selling_price × 6%（屬變動費，放分母）
- 售價 > 1000：FSS = NT$60 fixed（屬固定費，放分子）

逆推定價時需要迭代或分段計算，因為 FSS 是否用 fixed 取決於最終售價，而最終售價又依賴 FSS 類型。
建議實作：先用 pct 計算一次得到初估售價，再判斷 FSS 模式，再重算一次。

## 7. 最核心的用戶體驗原則（來自規劃文件）

> 「操作者應該能在幾秒內建立一個商品，然後再決定是否深入。」
> 「不要強迫操作者在建檔時就回答他還不知道的問題。」

這兩句是整個重構的核心哲學，所有 UX 決策都應回歸這兩個標準判斷。
