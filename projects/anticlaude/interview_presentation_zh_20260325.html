AntiClaude / AITOS / Flow Lab

我把 AI 協作、工作流審核、內容營運與電商決策整合成一套可操作的全端系統。 核心不是單點功能，而是把真實工作流程做成可追蹤、可審批、可維護的 operator platform。

Problem

我想做的不是聊天介面，而是一套可控的工作系統。

  • AI 可以生成很多東西，但沒有治理與審核。
  • 電商定價與採購常依賴手算或試算表。
  • 內容、通知、審核、報表與操作分散在不同工具。
  • 使用者很難知道現在最該做什麼。
目標是把 AI、workflow、approval、內容、電商決策，整合成一個真正可以操作的 operator system。

Architecture

從 prompt 和頁面，收斂成 workflow 與 domain logic

API 層

FastAPI 路由，負責 chat、review、workflows、ecommerce、Flow Lab、Figma、reports。

Runtime 層

workflow runs、tasks、approvals、events、checkpoints、resume / pause。

Domain 層

media 與 flow_lab 商業邏輯逐步分離，避免所有規則都塞在 UI。

Flow Lab

把電商從試算表，做成決策引擎

Bottom-Up 定價

  • 從採購成本與落地成本出發
  • 套用 Shopee 費率與毛利目標
  • 推回建議售價

Top-Down 採購天花板

  • 從市場價出發
  • 反推出最大可接受採購成本
  • 幫助 1688 / QQL 議價與選品
這讓 Flow Lab 不只是算帳，而是幫操作者決定：能不能做、怎麼賣、能不能談得下來。
Reliability

我不只做功能，也修復真正會傷害信任的問題

通知 spam

用 SQLite-backed dedup 擋掉重複 LINE 通知。

測試 publish 洩漏

修 dual publish 測試路徑，避免測試字串碰到真 Threads 發文。

Bundle runtime 修復

修 schema / query mismatch，讓組合建議路由恢復穩定。

Build 綠燈。 測試綠燈。操作信任感被救回來。
